# 阿迪达斯第三方标签 RFP

**来源**：Request for Proposal — Third party tags, adidas (Sports) China Ltd.
**形式**：DOCX，约 15 页
**发布时间**：2026 年 3 月 5 日
**截止时间**：2026 年 3 月 13 日

---

## 文档定位

本文件是阿迪达斯中国发起的第三方数据标签能力采购招标书（RFP），面向候选供应商征集解决方案。最终中标方为秒针/明略科技。本文件代表**需求方视角**，与秒针提案文件互为对照——RFP 定义了"要什么"，秒针提案回答了"怎么做"。

---

## 背景与痛点

阿迪达斯目前依赖内部一方数据（交易历史、站内行为、基础人口属性）进行会员分层和营销激活。核心问题：

- **会员总量大，但数据空洞**：8,700 万+会员，其中仅 14% 具备基础人口属性（姓名、性别、城市）
- **现有补全速度严重不足**：当前方法每年仅能补全 53 万会员基础档案，距离目标 1,000 万相差 19 倍
- **视角盲区**：内部数据仅覆盖会员在阿迪达斯生态内的行为，无法看到他们在外部的兴趣、意图、生活方式

---

## 项目目标

1. 为 **1,000 万会员**增补外部数据标签，丰富会员画像
2. 构建 360° 客户视图，支持精准营销和精细化会员运营
3. 将增补后的会员档案同步至零售门店，支持导购个性化服务
4. 升级按品类维度的会员数据看板，驱动渠道/BU 决策

---

## 六大交付要求（D1–D6）

### D1 — 外部标签采集与匹配
- 提供涵盖人口属性、生活方式、购买意向、心理特征等维度的标签目录
- 对 1,000 万阿迪达斯会员执行安全合规的身份匹配
- 提供数据来源、方法论、更新频率的完整文档

### D2 — 会员档案增补与 360° 视图构建
- 将增补属性集成进统一会员档案系统
- 与现有会员数据结构保持分类对齐（taxonomy alignment）
- 提供描述增补后会员群体特征、品类亲和度和行为模式的分析报告

### D3 — 零售与全渠道数据同步
- 支持准实时或批量同步增补属性至门店服务工具
- 提供 API 或文件集成规范、技术文档和测试支持

### D4 — 看板升级与分析能力建设
- 在现有看板中集成增补数据，支持品类级会员洞察、渠道规划、分层精细化、Campaign 和生命周期定向

### D5 — 数据质量、安全与合规
- 建立覆盖数据质量检查、匹配准确率报告、持续监控的治理框架
- 满足《个人信息保护法》（PIPL）及阿迪达斯内部数据保护政策
- 支持算法备案/登记及中国法律规定的其他监管程序

### D6 — 项目管理与文档
- 指定项目负责人，按阿迪达斯标准节奏（每周/双周例会）执行
- 交付完整文档：实施计划、数据字典、接口规范、QA 报告、培训材料
- 提供项目总结报告，包含成果、经验与持续优化建议

---

## 技术要求（硬指标）

| 指标 | 要求 |
|------|------|
| 系统可用率 | ≥99%（计划内维护除外） |
| 身份匹配准确率 | ≥80% |
| API 响应时间 | P95 ≤ 500ms |
| 变更提前通知 | ≥10 个工作日 |
| 数据覆盖范围 | 10M+ 会员，批量初始化 + 增量同步 |
| 合规要求 | PIPL + 阿迪达斯内部数据保护政策 + 算法备案 |

**关键安全红线：**
- 阿迪达斯数据不得被第三方（包括承包商和关联方）在约定范围外访问或使用
- 阿迪达斯数据不得用于模型训练、画像分析或合同范围外的任何用途
- 合同终止后，支持完整的数据删除和审计日志

## 阅读要点（与秒针提案对照）

本 RFP 定义了阿迪达斯的需求框架，对照[[adidas/sources/miaozhen-3rd-party-tags-proposal-20260313|秒针提案]]可以看出：

- RFP 要求匹配率 ≥80%，秒针承诺 ≥80%（行业均值 60%）
- RFP 要求标签涵盖人口、兴趣、购买意向、心理特征，秒针提供 5,000+ 标签 / 42 大类覆盖
- RFP 要求 API P95≤500ms，秒针在提案中有对应 SLA 承诺
- RFP 明确禁止将阿迪达斯数据用于训练，秒针方案采用隐私计算（PSI+TEE）规避数据直接交换
