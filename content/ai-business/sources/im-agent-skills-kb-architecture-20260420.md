# IM + Agent + Skills + Knowledge Base：新一代企业软件架构

**来源**：半瓶水笔记（公众号）
**发布时间**：2026-04-20
**原文链接**：https://mp.weixin.qq.com/s?__biz=MzYyMTI3ODg4OA==&mid=2247484614&idx=1&sn=3464245f69c3d9f237e200985c27a096

---

## 摘要

作者基于多个企业 Agent 落地项目的经验，提出了一套重写企业软件底层逻辑的架构主张：以 IM 为统一入口、Agent 为调度核心、Skills 为可编排的业务积木、知识库为持续学习记忆，将原本散落在多个系统中的数据、规则、上下文连成一张活的网。

核心判断：当 GPT-5 成为公共资源，算法差异化消失，**企业的真正护城河在于谁的知识飞轮转得最快**。

---

## 核心要点

### 企业软件五十年的三次失败

过去五十年的企业软件演化，是三次大规模试错：

1. **以"流程"为中心**（ERP/OA时代）：流程固化带来系统僵化，数据跟着流程产生孤岛
2. **以"数据"为中心**（数据中台时代）：结构化数据只是企业知识的很小一部分，上下文、决策依据无法被捕获；中台建设周期长（2-3年）、成了新孤岛
3. **以"工具"为中心**（SaaS时代）：Okta数据显示企业平均使用80+个SaaS工具，孤岛变群岛，集成成本反超单一平台价值

**第四条路：以"知识"为中心**——让知识能被自动化地理解、使用、迭代。

### IM 为什么是正确的入口

四层逻辑：
- **使用频率**：IM日活渗透率超过90%，是唯一"总是打开的"应用
- **上下文完整性**：最有价值的工作指令产生于IM对话，天然携带完整上下文（项目、时间、关注者）
- **协作即输出**：Agent输出结果在IM中呈现，直接进入团队讨论，形成"输出即协作"闭环
- **跨系统粘合**：真实业务流程的跨系统协调（CRM→OA→ERP→项目管理），大量发生在IM的人工协调中

关键设计选择：Agent 不应是 IM 里的一个"机器人账号"，而应是 IM **底层协议级别的整合**——前者是工具，后者是基础设施。

### 四层架构概览

| Layer | 名称 | 核心功能 |
|-------|------|---------|
| Layer 0 | 交互层（IM） | 人机接口 + 协作输出渠道 + 上下文采集节点 |
| Layer 1 | Agent 层 | 意图理解 + 任务规划 + 动态编排 + 人工确认机制 |
| Layer 2 | Skills 层 | 业务能力原子化，把系统能力翻译为业务语义 |
| Layer 3 | 系统原子能力层 | 通过 MCP/API Gateway 把 ERP/CRM/OA 能力暴露给 Skills |
| Layer 4 | 企业知识库 | 持续自动积累的动态记忆系统，而非静态文档库 |

### Skills 的本质

Skills 是一种全新的**业务规则解耦方式**：
- 传统做法：业务规则写在代码逻辑/BPM流程引擎里，高度耦合，改一个规则需要改代码+测试+发布
- Skills 做法：业务能力原子化，由 Agent 在上下文中动态编排，业务规则可用自然语言描述和调整

从"流程驱动系统"到"**意图驱动编排**"的范式转移。

最接近工业化实现的是 Anthropic 2024年11月发布的 **MCP（Model Context Protocol）**——Skills 可理解为"企业语境下的 MCP Tools"。

### 知识飞轮的飞轮效应

每次 Agent 成功完成任务 → 知识库更丰富 → Agent 下次处理同类问题成功率更高 → 员工更信任 Agent → 使用频率更高 → 知识积累更快。

知识库核心特性：
- **多模态输入**：文档、表格、IM对话、会议纪要、代码注释
- **知识图谱增强**：实体关系显式建模
- **版本管理**：知识时效性感知
- **主动入库**：Agent 执行后自动将成功方案摘要写回知识库
- **知识审核**：关键知识需人工确认后进入"高信度层"

### 市场竞品分析

| 玩家 | 定位 | 核心局限 |
|------|------|---------|
| 飞书 | 最接近完整愿景 | Agent视野边界是飞书平台内；Skills体系尚未成熟 |
| Salesforce Agentforce | CRM域内的垂直架构 | 以"客户数据"为中心而非"企业全知识"；商业模式决定天花板 |
| Microsoft Copilot | 规模优势但生态绑定 | 效果依赖M365数据治理质量；非M365数据处理能力有限 |
| Coze / Dify | 个人/团队级构建工具 | 缺乏企业级IAM/数据隔离/审计/私有化部署；知识库是静态索引 |
| Glean | 企业搜索最强 | 只能"找到答案"，不能"帮你做这件事" |

**空白地带**：无任何玩家真正实现"IM统一入口 + 通用Agent跨系统执行 + Skills业务规则可编排 + 知识库持续自主积累"的完整闭环。

### 信息化的核心矛盾

50年始终未解：**企业的真实运营上下文（Context）无法被系统完整捕获和利用。**

- ERP 捕获了交易数据，但没有捕获决策依据
- CRM 捕获了客户信息，但没有捕获销售对客户的直觉判断
- 真正有价值的知识存在于：一封邮件的背景、一段微信群的临时方案、老销售脑子里的客户关系图谱

IM + Agent + Skills + KB 的根本主张：**第一次认真地把"系统缝隙里的上下文"作为一等公民来对待**。
