# To Human / To Agent 新市场坐标系

## 定义

Agent 时代的新市场分类框架，用"服务对象是人还是 Agent"替代传统的 To B / To C 划分。

- **To Human**：产品/服务的最终消费者是人类用户（个人或组织）
- **To Agent**：产品/服务的客户是 AI Agent——Agent 自主开户、调用、付费、消费资源

## 运作方式

**为什么 To B/To C 失效**：
- SaaS 时代 PLG vs. Enterprise 的界限清晰；而 AI 工具普及顺序是"个人先付费、公司 IT 预算追入"
- 同一产品同时服务"个人工作流"和"组织工作流"，To B 与 To C 的传统边界已模糊
- 统一特征：to Prosumer（知识工作者）

**Agent 成为独立经济主体**：
- Cloudflare：允许 Agent 自主创建账号、开通付费订阅、注册域名、拿到 API token 部署代码 → Agent 第一次以独立客户身份进入云服务商客户体系
- Stripe：为 Agent 提供身份与支付中枢
- Anthropic：通过 MCP/Skills/computer use 将 GUI 转为 CLI 或 API，把软件操作权交给 Agent
- Cloudflare pay-per-crawl：对 AI 爬虫单独计费 → Agent 既是付费客户，也是被独立计费的流量主体

**基础设施重构方向**：
整个市场正在为 Agent 打造新的生产环境，典型特征：Agent 原生索引（llms.txt）、Agent 身份系统、Agent 支付轨道。

## 为什么重要

- 产品设计需要 agent-native 思维：DAU、ARPU、MAU 等互联网指标不再适用
- 投资价值判断需要新标尺：同一款产品面向人类和 Agent 的定价逻辑、增长路径完全不同
- 开创了新的商业机会：Stripe/Cloudflare 等基础设施公司正在捕获"Agent 经济"的底层份额
