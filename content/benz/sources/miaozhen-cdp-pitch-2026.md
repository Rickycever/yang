## 行业痛点定义（五大困局）

| # | 痛点 | 表现 |
|---|------|------|
| 1 | 数据分散 | 消费者数据散落多渠道，缺乏统一视图 |
| 2 | 身份模糊 | 同一用户多个 ID，无法识别完整行为路径 |
| 3 | 决策滞后 | 跨品牌圈选需人工合并，窗口期错失 |
| 4 | AI 难落地 | AI 功能外挂叠加，无法基于一致数据决策 |
| 5 | 数据安全 | 跨系统数据流转，合规压力持续升高 |

|------|------|------|
| **统** | UNIFY | 全渠道数据汇聚 · OneID 身份融合 · 消费者 360° 视图 |
| **活** | ACTIVATE | 精准圈选实时分发 · 对接 CRM/MA/投放 · 标签人群洞察 |
| **智** | INTELLIGENT | AI 原生决策引擎 · NeuroMA 自动营销 · 大模型+知识图谱 |

## 核心差异化：行业知识底座

明略 AI 圈人的关键竞争壁垒不是模型，是**行业知识底座**：

> "LLM 能理解'找4-7月龄的妈妈'，但需要知道'米粉是4月龄强信号、维生素D是全段噪音'——这些不在模型里，在行业经验里。"

| | 竞品（裸 AI 套壳） | 明略（AI + 行业知识底座） |
|---|------|------|
| 行业知识 | 通用模型无行业知识 | 信号→意图映射库（母婴/快消/汽车） |
| 规则校验 | 圈选规则无从校验 | 跨 Campaign 经验自动提炼沉淀 |
| 可解释性 | 错误信号无法识别 | 领域知识可审计、可迭代、可解释 |
| 效果稳定性 | 效果不稳，信任难建 | AI 规则与专家规则吻合度 **90%+** |

**母婴行业示例：**
- 米粉/辅食碗 = 4月龄强信号 ✓
- 维生素D = 全段噪音 ✗（不能用于年龄判断）
- 磨牙棒 = 6月龄信号 ✓

## 技术架构：AI 原生五层全栈

全栈自研，无第三方依赖：

| 层级 | 技术组件 |
|------|------|
| 应用层 Application | K8S微服务 · Kafka · PostgreSQL · Redis |
| 智能层 Intelligence | Milvus 向量库 · Neo4j 知识图谱 · LangGraph AI Agent |
| 计算层 Compute | Flink 流批一体 · StarRocks 亿级OLAP 亚秒响应 |
| 存储层 Storage | Hive · S3/HDFS · MinIO 私有化对象存储 |
| 接入层 Ingestion | CDC · Kafka · DataX · API 多源接入 |

**安全保障三件套：**
- 全程 AES 加密（数据始终密文流转）
- KMS 密钥管理（统一密钥全链路可控）
- 私有化部署（无需公有云，完整闭环）

---

## 三大结构性差异化优势

明略强调是**结构性优势，而非功能性优势**：

**01 AI 原生，不是 AI 外挂**
- AI 与数据共享同一事实层
- 竞品：人工写规则 → 导出人群包，3-5天
- 明略：输入自然语言 → 实时生成规则+量级，**分钟级交付**

**02 安全优先架构**
- 全程 AES 加密 + KMS 管理，数据始终密文流转
- 强监管行业（汽车/金融/医疗）优选

**03 私有化全栈能力**
- 无需公有云，完整闭环部署
- MinIO + Keycloak + K8S
- 从接入到 AI，无第三方依赖

## 三阶段落地路径

| 阶段 | 内容 | 业务价值 |
|------|------|------|
| Phase 1 — 数据基础 | 数据接入 & ETL · OneID 融合 · 消费者 360° 视图 | 数据资产盘活，消费者全貌可见 |
| Phase 2 — 营销激活 | 标签工厂建设 · MA 平台对接 · 精准圈选分发 | 营销效率提升，精准触达落地 |
| Phase 3 — AI 驱动 | NeuroMA AI 圈人 · ChatBI 智能分析 · 智能旅程自动化 | AI 驱动自动化，分钟级圈选，降本增效 |

**风险控制机制：** 完整迁移 SOP · 分阶段交付 · Rollback Plan · 业务方全程参与验收
