# RTA 程序化投放

## 定义

RTA（Real-Time Audience）是品牌方自有数据系统与媒体平台竞价系统实时对接的投放模式。媒体在拍卖某个广告位时，会实时查询品牌方的 RTA 前置机，由品牌方决策"这个用户是否是我的目标人群"，再决定是否出价及出价金额。

## 运作方式

**流量入口**：
- 合约流量（PD/PDB）：预购固定位置
- 竞价流量（RTA）：实时竞价，品牌自主决策

**全域 AI 智能投放平台架构**：
```
秒针监测 → 前置机 → 明略CDP → 明略标签 → 竞价平台
```

AD Server 策略机提供：频次服务 / 人群服务 / 创意服务 / 出价服务 / 控量服务

**三大核心优势**：
1. **明略标签加持**：标签 + 算法 + ID转译 + 异常过滤
2. **AI 自动智能出价**：根据用户价值评分动态调整出价
3. **全域频次控制**：跨媒体统一控制同一用户的广告曝光次数，避免骚扰

**覆盖**：全终端（PC/MO/OTT）× 全广告形式（Banner/贴片/暂停/信息流）× 四大主流媒体（腾讯系/字节系/快手系/百度系）及中小媒体

**业务价值输出**：
- 控制成本（CPM/CPC）
- 触达最大化（TA Reach）
- 高拉新率
- 高转化率（促活 ROI）

## 为什么重要

传统广告按人群包批量购买，属于"猜测式投放"；RTA 是"实时验证式投放"——每次曝光前都经过品牌自己的数据系统判断，大幅减少无效曝光浪费。

对中欧这类客单价极高、目标人群极小众的客户，精准 > 规模。RTA 让品牌对每次广告请求都有控制权，与 CDP 的标签和 Lookalike 模型直接打通，是数据资产变现的核心出口。
