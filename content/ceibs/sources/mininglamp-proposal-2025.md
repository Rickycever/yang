# 中欧商学院数字化营销能力建设提案（明略科技，2025.07）

## 摘要

明略科技为中欧国际商学院（CEIBS）提供的数字化营销能力建设整体方案，聚焦 EMBA/MBA 招生场景。方案围绕四大业务问题（精准拉新 → Leads 分层管理 → Leads 高效分配 → 有效触达）展开，提出三阶段落地路线，核心产品为明略 CDP + MA + 企微 SCRM + 秒针 SaaS DMP。

## 核心要点

### 四大业务问题

| 模块 | 核心动作 |
|------|---------|
| 精准拉新 | 精准广告引流、直播/种草引流 |
| Leads 分层管理 | 优质 Leads 分级、非目标 Leads 识别 |
| Leads 高效分配 | 线上分配、用户标签共享、流程节点跟进 |
| 有效触达 | 广告、企微、短信、电话、邮件 |

### 精准拉新四大能力

**1. 360° 用户画像**
- 明略三方标签体系：9 个类别、1000+ 标签（人口属性、人生阶段、兴趣爱好、行业偏好、触媒偏好、设备信息、APP 倾向、LBS、消费需求、OTT 标签）
- 通过安全岛（Safe House）环境进行隐私合规的数据融合，符合 PIPL 要求

**2. LBS 定向能力**
- 目标场域：CBD 商务区、别墅区/度假村、中高端汽车 4S 店、文化馆/展览馆、技术峰会、网球馆/高尔夫球场
- 对比媒体平台 LBS：转化率提升 **50%**
- 时空四维控制：人群 × 时间 × 城市 × 精准空间

**3. Lookalike 高价值人群拓展**
- 流程：种子包转译（手机号 → DID） → 秒针 ID 增强 → 正/负样本构建 → 机器学习（逻辑回归/GBDT/FMM/DeepFM）→ Lookalike 放大 → 人群包投放
- 联合计算架构：品牌一方数据 + 秒针大库标签 + 个推三方数据，在安全岛中计算

**4. RTA 广告投放**
- 全域 AI 智能投放平台：合约流量（PD/PDB）+ 竞价流量（RTA）
- 全终端（PC/MO/OTT）× 全广告形式 × 全媒体（腾讯系/字节系/快手系/百度系）
- 核心优势：明略标签加持 + AI 自动智能出价 + 全域频次控制

**5. 腾讯社交关系网分层触达**
- 高意向人群（既有学员）→ 企微互动，调动关系网络
- 中等意向人群（高潜人群）→ 朋友圈广告精准曝光
- LBS 定向人群 → 区域定向拓展

### 秒针×个推 ID Mapping 服务
- 合资公司：明日数据（秒针 + 个推）
- Device ID → PN-ID 匹配率：**80%+**（行业基准 60%）
- 覆盖：95%+ 网民，90%+ DMP 识别率，10B 峰值日志处理

### Leads 分层管理

**Leads 用户旅程五阶段**：
1. **认知阶段**：被动接触社交内容（焦虑现状，缺乏信任）
2. **兴趣阶段**：报名线上活动、加企微（验证"是否值得投入"）
3. **评估阶段**：1v1 咨询、对比竞品长江/复旦（理性权衡）
4. **转化阶段**：完成申请、面试、缴费（需"最终确认感"）
5. **留存与裂变阶段**：参与校友活动、推荐潜在学员（品牌拥护者）

**四类标签体系**：
- 基础属性标签：35-45 岁、高管、金融/创投/科技行业
- 行为兴趣标签：内容好奇、职业转型、行业前沿、EMBA 对比
- 意向与转化标签：原访阶段、流量转化、高潜转化、竞争申请
- 校友裂变标签：参与度、KOL 投发、转介绍达人、高互动分享者

**Leads 分级输出**：优质 Leads（Hot）/ 普通 Leads（Warm）/ 非目标用户（Cold）

### OneID 三种打通方式

| 方式 | 特点 | 适用场景 |
|------|------|---------|
| 优先级打通 | 减少无效绑定，降低计算复杂度 | 动态复杂 ID 关系 |
| 全关联打通 | ID 间无主次，串联更多数据源 | 需要最大覆盖 |
| 算法打通 | 非确定性，不受制于强账号体系 | 预测更多数据源 |

底层：图计算 + Redis 存储，实时增量计算

### Leads 跟进与触达

**企微 SCRM**：
- CDP × 企微打通，标签/人群实时共享
- 侧边栏：360° 画像 + 客户轨迹 + 素材库 + 快捷回复
- MA 派发员工任务：标签定位目标人群 → 任务下达 → 员工执行 → 结果反馈
- 智能继承：员工离职/转岗，客户资产（画像、历史记录）一键转移

**触达渠道**：微信、企微、Email、短信、抖音、广告（支持事件触发 + 定时）

### 三阶段实施路线

| 阶段 | 目标 | 所需能力 |
|------|------|---------|
| 阶段一 | 基于中立国，尝试精准投放 | 一方数据对接安全岛 + SaaS 版 RTA 投放 |
| 阶段二 | 建立基础 CDP，完善标签体系 | CDP 搭建 + 营销流程 SOP + Leads 分级模型 |
| 阶段三 | "企微+CDP" 覆盖用户拉新服务全流程 | 企微与 CDP 打通 + 服务流程跟踪定制开发 |

### 明略 CDP 能力矩阵
- **数据治理层**：数据底座（ETL）+ 数据模型（OneID/Profile/Event/Transaction）+ 数据质量
- **消费者洞察层**：标签构建（指标/行为/自定义）+ 人群应用（AIPL/RFM/Lookalike）+ 人群画像
- **分析层**：用户行为分析（漏斗/路径/归因/留存）+ Dashboard + SmartBI
- **营销自动化层**：MA 营销旅程 + 触点管理 + A/B Test + 内容/权益推荐
- **CDP API 服务**：人群包推送/创建 API + 标签查询 API + 消费者 360 查询 API
