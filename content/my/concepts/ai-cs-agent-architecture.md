# AI客服智能体架构（单一Agent + MCP）

## 定义

针对连锁餐饮/零售等"单点高并发客服"场景的 AI Agent 架构模式。采用**单一智能体 + MCP（Model Context Protocol）**方案：核心智能体负责意图识别与工作流调度，通过 MCP 协议连接订单系统、知识库、会员系统等多个后端服务。

海底捞实现中使用的 Agent 平台为 **HiAgent**（明略科技/明胜品智自研）。

## 运作方式

**信息流向**：
```
多渠道消息（企微/电话/小程序）
  ↓
聚合聊天平台（统一接入，自动封装元数据：时间戳/门店ID等）
  ↓
意图识别层（订餐/排号/咨询/客诉/会员 → 分流进入对应工作流）
  ↓
AI客服智能体（HiAgent）
  ↓ MCP 协议
[订餐接口] [排号接口] [门店知识库] [FAQ知识库] [活动知识库] [会员系统]
  ↓
海底捞开放平台（最终操作执行）
```

**提示词框架**（6段式，控制 Agent 行为边界）：
```
# 角色       → 定义 Agent 人设与授权范围
# 工具说明   → 每个工具的用途与调用条件
# 你的任务   → 核心业务目标
# 执行流程   → step-by-step 标准流程
# 注意事项   → 异常处理规则（转人工条件）
# 限制输出   → 输出格式与禁止项
```

**人工保底机制**：满足以下任一条件时 AI 触发转人工信号：
- 用户情绪异常（识别到投诉/愤怒语气）
- 信息模糊无法澄清
- 系统调用失败
- 额外需求超出工作流覆盖范围

## 为什么重要

**与多 Agent 架构的对比**：
- 单一 Agent 工作流简单线性，延迟低，适合"单点服务"（客服是一对一交互，不需要并行 Agent 协作）
- MCP 协议解决多系统连接的标准化问题，避免为每个系统写定制集成
- 实际降本效果：同等服务质量下，10人客服团队可缩减至2人

**可迁移性**：该架构适用于任何具备以下特征的业务：
1. 明确的意图分类（有限的业务场景）
2. 后端系统有标准接口
3. 流程标准化程度高
