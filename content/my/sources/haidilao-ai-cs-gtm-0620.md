# 海底捞AI客服智能化方案 & GTM复制路径

## 摘要

明胜品智/明略科技出品。以海底捞为标杆案例，完整呈现从"人工应答"到"AI智能体驱动"的客服流程重构全貌，包含：技术架构（单一智能体+MCP）、动态知识库机制、提示词框架、工作流设计，以及四大模块的跨行业复制路径。文件编号：0620。

## 核心要点

### Part 1：海底捞AI客服场景与业务成效

**痛点**：传统订餐/排号依赖人工，响应慢、质量波动、门店体验不一致。

**目标**：基于大模型构建AI客服系统，实现服务流程标准化、自动化、智能化。

**四大价值**：

| 维度 | 说明 |
|------|------|
| 提效 | 7×24小时服务，响应一致性↑ |
| 降本 | 同等服务质量下，10人团队可缩减至2人完成核心任务 |
| 可扩展 | 适配单店/区域/总部多层客服策略 |
| 可复制 | 流程模块化设计，具备跨行业快速复用潜力 |

**AI客服覆盖场景**：
- **订餐/排号**：多轮对话自动收集信息（姓名/手机号/人数/时间/特殊需求），调用系统接口完成操作
- **客诉/投诉**：识别情绪异常，自动转人工
- **发票开具**：识别订单号，自动触发开票流程
- **门店/菜品/活动咨询**：调用对应知识库精准回复
- **会员查询**：积分/卡券/权益智能响应

---

### Part 2：客服智能化方案架构

**架构设计**：单一智能体 + MCP 方案

```
用户消息
  ↓
聚合聊天平台（多渠道：企微/电话/小程序）
  ↓
后台自动封装（时间戳/门店ID等）
  ↓
AI客服智能体（HiAgent，意图识别 + 流程引导 + 工具调用）
  ↓ ↓ ↓ ↓ ↓ ↓
订餐系统  排号系统  门店知识库  活动知识库  FAQ知识库  其他
  ↓
海底捞开放平台服务
```

**设计特点**：
- 工作流简单线性，解决多系统连接问题
- 单点服务，无需多 Agent 协作，保障高响应速度
- 人工保底机制：AI无法处理时自动转人工

**提示词框架**（6段式）：
```
# 角色
# 工具说明
# 你的任务
# 执行流程
# 注意事项
# 限制输出
```

### Part 3：动态知识库机制

知识库分层：
- **门店信息知识库**：门店列表/地址/所属关系/营业时间/电话/周边设施（飞书文档，门店自行维护，AI定期同步）
- **FAQ知识库**：过往+当前常见问题（订餐流程/退换菜政策/会员权益）
- **活动/卡券知识库**：当前促销活动与规则

知识库架构：
```
用户提问 → AI客服智能体 → 门店知识库智能体
                            ↕（查询指定门店知识库）
                         营食亿门店知识库服务
                            ↕（知识库同步）
                         飞书文档（门店自维护）
```

通过 Agent + AI workflow编排 + RAG检索 + 模型管理，支持"千人千聊"智能体台。

### Part 4：平台选型 —— Dify vs Coze

| 维度 | Dify | Coze（字节） |
|------|------|------------|
| 部署方式 | 支持私有化部署 | 基于字节平台，无法私有化 |
| 数据管理 | 自主管理，数据完全归品牌方 | 数据需交由 Coze/字节 |
| 模型接入 | 支持自定义模型 | 基于字节模型平台 |
| 数据安全 | 可控性强，适合数据安全敏感企业 | 合规性存在隐忧 |
| 成本 | 私有化部署成本略高 | 使用现成服务，成本较低 |

**结论**：对数据主权要求高的企业（如连锁餐饮）优先 Dify；快速验证/低合规压力场景可选 Coze。

### Part 5：GTM 方案复制路径

| 模块 | 适用场景 |
|------|---------|
| 聚合聊天与人工协同机制 | 拥有客服团队的连锁型企业 |
| 意图识别 + 流程引导 | 快速复制到快餐/电商等客服场景 |
| 动态知识库机制 | 适配区域化、个性化门店运营需求 |
| 模块化架构 + 提示词工程 | 可迁移至任意业务流程标准明确的企业 |

### Part 6：真实对话示例

**订餐场景**（多轮信息收集 → 系统调用）：
> 用户："我要订餐" → AI收集：门店/时间/人数/特殊需求/手机号 → 订餐成功确认

**排号场景**：
> 用户："现在需要排队吗，我们4个人" → AI引导姓名+手机号 → 排号成功（小桌8号）

**客诉场景**（自动转人工）：
> 用户反馈账单错误 → AI获取手机号 → 触发转人工信号 → 门店小伙伴跟进

**活动咨询**（规则知识库调用）：
> 用户："学生折扣可以跟小程序折扣叠加吗？" → AI：不可叠加，并说明规则
