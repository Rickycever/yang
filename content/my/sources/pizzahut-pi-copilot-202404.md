## 产品定义

**π（PizzaHut Intelligence）** — 必胜客营运超级助手，面向餐厅经理（RGM）的AI Copilot系统，以COL%（人力成本比率）管理为切入点，通过"数据找人"替代"人找数据"，辅助RGM定位机会点并采取行动。

---

## 核心问题：RGM的灵魂三问

```
餐厅COL有没有提升空间？
        ↓
机会点在哪里？
        ↓
我能做什么？可以做什么？
```

---

## 产品架构：Copilot + Agents

```
超级助手 Copilot（π）
├── 多轮会话，澄清用户真实意图
├── 拆解解题步骤和工具需求
├── 监督不同Agent完成工作
└── 汇总归纳后反馈用户
        ↓
┌───────────────────────────────────┐
│  运营管理知识Agent                 │
│  身份：资深餐厅运营专家             │
│  工具：知识库/知识提取/标签化       │
├───────────────────────────────────┤
│  数据查询分析Agent                 │
│  身份：数据分析师                  │
│  工具：数据查询/图表生成工具        │
├───────────────────────────────────┤
│  BSC生成Agent                     │
│  身份：PPT写作专家                 │
│  工具：PPT生成工具                 │
├───────────────────────────────────┤
│  更多Agents（管理分析/事务处理）    │
└───────────────────────────────────┘
```

**设计原则**：使用不同工具解决不同问题，Agent能力越聚焦效果越好。

## 两大技术支柱

| 支柱 | 具体内容 |
|------|---------|
| **数据科学** | 梳理管理框架、定义管理模型（标杆/理想/平均/排序）、拆分管理层级（成本结构→工时→人效→人员质量）、探索到人到岗的直接定量分析方法 |
| **AIGC** | 专属助手全新交互范式、"数据找人"输出重点、知识萃取/知识问答、数据问答、千店千面输出 |

---

## 两大底座

**数据底座**：
- 复用当前PH营运数仓
- 数据宽表 / 数据指标 / 数据建模
- 数据清洗 / 数据汇算

**知识底座**：
- 复用当前YUMC知识库
- QA库 / 知识库 / 指标库 / 分析方法论
- 知识检索 / 知识萃取 / 标签提取

---

## 四大价值场景

| 场景 | 价值描述 |
|------|---------|
| 专属导师 | 通过运营知识提炼，构建新人成长加速器 |
| 个性化经营诊断 | 建立单店级别经营诊断助手，拉升门店管理平均水平 |
| 一站式数据视图 | 降低餐厅数据门槛，整体经营数据视图 |
| 事务效率提升 | 集成数据查询/BSC PPT生成/流程跟进等事务型工具 |

---

## COL模块交互流程

1. 口袋经理首页展示餐厅重点管理机会（营运One Page）
2. 检测到COL有机会点时，引导进入π
3. 展现实际Sales下与理想COL%的差异，拆解成本结构，分析最大机会点
4. 可视化图表展示数据洞察
5. 通过"关联问题"和"多轮对话"定位机会点（到场景/到岗/到人）

---

## 迭代路径

```
收集种子用户问题清单 → 梳理管理经验&方法论 → 数据探索"A-HA时刻"
        ↓
"基本盘"构建（Copilot功能 + 数据底座 + 知识底座）
        ↓
迭代演进（用户反馈 + 大模型能力升级 + 更多数据 + 新A-HA时刻）
        ↓
餐厅运营系统新范式
```
