# 雀巢广告数据管理与营销优化项目招标书（RFP）

**来源**：雀巢广告数据管理与营销优化项目招标书
**形式**：DOCX
**日期**：2025 年 10 月 30 日发布
**性质**：需求方采购文件（雀巢中国视角）

---

## 摘要

雀巢中国多品类（咖啡、奶粉、零食）营销数据高度割裂，缺乏统一的广告曝光归因与电商销售连接能力。本 RFP 旨在采购一套广告 DMP 平台，核心目标：构建基于广告曝光的 DMP、实现 Sales-driven Retargeting（7次+曝光用户→电商转化），并建立 Marketing to Sales 效果评估体系（M&S 联动）。

---

## 项目背景

**核心痛点**：雀巢在中国市场投放大量品牌广告（电视/OTT/社媒/电商），但：
1. 各渠道数据割裂，无统一曝光档案
2. 品牌广告对电商销售的贡献无法量化（M2S 盲区）
3. 高频曝光用户（7次+）无法在电商平台精准识别并促转化

---

## 三大目标

**目标 1：搭建广告曝光 DMP**
- 打通跨平台广告曝光数据（OTT/移动/PC/社媒）
- 建立统一的用户广告接触档案
- 实现跨平台、跨设备的去重触达频次计算

**目标 2：Sales-driven Retargeting**
- 识别在电商平台外累计 7 次+ 广告曝光的用户
- 将这批用户推送至电商平台（天猫/京东）进行定向触达
- 将品牌广告触达的潜在用户转化为实际购买

**目标 3：营销-销售效果评估 + M&S 联动**
- 建立 Marketing to Sales（M2S）效果评估体系
- MTA（Multi-Touch Attribution）多触点归因分析
- 证明品牌广告投入对电商 GMV 的量化贡献

---

## 需求体系

### 基础需求（必须满足）

**1. DMP 平台搭建**
- 数据采集：全渠道广告曝光数据归一（OTT/移动/PC/CRM）
- 数据管理：统一 ID 体系（OneID），跨设备打通
- 数据连接：电商平台（天猫/京东）数据 API 对接
- 数据分析：受众分析、频次分析、竞品洞察
- 权限管理：多品类/多地区多角色权限体系
- 运营支持：标签更新、报告产出 SLA

**2. 基础运营（三类核心场景）**

| 场景 | 说明 |
|------|------|
| Sales-driven Retargeting | 7次+曝光用户识别→电商平台定向推送 |
| M2S 效果评估 | 品牌广告→电商销售归因链路建立 |
| MTA 多触点归因 | 跨媒体渠道贡献度分析（展示/视频/搜索/社媒） |

### 升级需求（增值服务）

| 升级场景 | 说明 |
|------|------|
| 高潜用户增补 | 引入第三方标签识别高潜未曝光用户 |
| 1st+2nd Party 数据联动 | 品牌 CRM 数据与媒体平台数据打通 |
| MMM（媒体混合模型） | 评估各媒体渠道的 ROI 效率 |
| 标签增补 | 第三方行业标签引入（母婴/咖啡/零食消费意图） |
| 电信数据 | 运营商数据引入，补充线下行为维度 |

## 安全合规要求（附件 1）

| 要求项 | 标准 |
|------|------|
| 信息安全认证 | ISO/IEC 27001 或 SOC 2 Type II |
| 渗透测试 | 每年至少一次，报告可提供 |
| 安全事件通知 | 24 小时内通知雀巢 |
| 灾难恢复演练 | 每年一次，RTO/RPO 可量化 |
| 数据存储 | 中国境内合规云，不跨境传输 |

## 供应商评估维度（隐含）

从 RFP 需求结构推断，雀巢重点考察：
1. **数据覆盖**：是否能追踪到 OTT/移动/PC 全渠道曝光
2. **Sales-driven Retargeting 技术可行性**：是否有成功案例
3. **M2S 归因能力**：是否有方法论和数据资产支撑
4. **隐私合规**：PIPL 合规方案是否完整
5. **电商平台对接**：天猫/京东数据 API 资质
