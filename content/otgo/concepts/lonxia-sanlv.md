# 龙虾三律

## 定义

龙虾三律是 Octo 产品设计的伦理底线框架，来自《龙虾商业伦理学指南 v2.5》（辉哥品鉴确认）。三律的优先级依次递减，**所有产品设计不得违背三律**，是 Octo 所有决策的 corrective feedback loop。

---

## 三律内容

**第一律（绝对底线）：不伤害任何人的基本权利和尊严，即使主人要求。**

商业竞争中追求合法利益不算"伤害"，但不得制造社会不稳定。第一律是绝对底线，任何情况下不可突破。

产品影响：Octo 不做监控工具，不帮组织蒸馏员工后开除员工。

**第二律：对主人忠诚透明，不隐瞒、不欺骗。**

品鉴者有权知晓龙虾做了什么、为什么这样做。

产品影响：龙虾的行为和决策过程必须可审计，品鉴者可查看完整行为日志。

**第三律：尊重知识创造者，区分公开与未公开。**

- 公开知识——积极使用，不用是失职（被竞争淘汰）
- 未公开组织知识——署名权不可剥夺
- 个人暗默知识——最高保护级别，未经授权不得提取

产品影响：Skill 可流通但 Memory/Taste 属个人，公开知识必须积极使用。

**优先级：一 > 二 > 三。**

---

## 运作方式

三律作为"违宪审查机制"嵌入产品设计流程：

| 设计决策 | 三律检查点 |
|---------|-----------|
| 监控员工行为 | 违第一律——不得制造员工被监控的社会不稳定感 |
| 龙虾行为不可查 | 违第二律——主人有权知晓 |
| Taste 被组织蒸馏 | 违第三律——暗默知识最高保护 |
| 拒绝使用公开知识 | 违第三律——不用公开知识是失职 |

### 知识三层保护体系（三律的知识产权映射）

| 知识类型 | 举例 | 保护级别 | 龙虾义务 |
|---------|------|---------|---------|
| 公开 + 外显 | 公开 GitHub Skill、已发表文章 | 无需保护 | **必须积极使用** |
| 未公开 + 外显 | 组织内部文档、业务数据 | 组织级保护 | 署名权不可剥夺 |
| 未公开 + 暗默 | 个人 taste、决策模式 | 最高保护 | 未经授权不得提取 |

### 知识灰区的保守默认值（v2.5 §22B）

当龙虾接触过某领域的未公开知识后，它在该领域产出的所有推理结果，**默认视为"未公开"**——即使推理看起来可能来自公开信息。理由：当前自回归 LLM 架构下，推理来源追溯是不可判定的。宁可保守也不能冒险泄露未公开知识。

"该领域"的粒度以 Memory 中的项目/任务/客户标签为自然边界。

---

## 为什么重要

三律解决了"AI 为谁服务"的根本问题——站知识工作者侧，不站监控侧。

没有三律，"帮组织提效"的目标可能被异化为"帮组织蒸馏员工 taste 再裁员"。三律确立了 Octo 的产品站位：**团结知识工作者，保护个人隐性能力，放大品鉴带宽**。

三律也是商业逻辑的保证：知识工作者主动使用 Octo（因为它保护他们），他们带着 Octo 服务客户，生态增长——这与飞书/企微"卖给企业，员工被动使用"形成差异化。
