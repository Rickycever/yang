# OpenClaw 点火计划成果分享会 (2026-03-17)

**Date:** 2026-03-17 | **Duration:** 7+ hours | **Teams:** 24 | **Attendees:** 60+
**Host:** 吴明辉 (辉哥)

OTGO's first internal lobster showcase: 24 teams demonstrated real-world agent deployments across marketing, R&D, finance, HR, and government sectors. This session crystallized the universal playbook, surfaced recurring bottlenecks, and produced the first inventory of commercially validated lobster use cases.

<div class="zh-trans">OTGO 首次内部龙虾落地成果展示：24 支队伍跨营销、研发、财务、HR、政府等场景实战分享。本次会议沉淀了通用打法、梳理了共性卡点，并产出了首批经商业验证的龙虾应用清单。</div>

## Universal Patterns (Present in 5+ Teams)

### 1. Pipeline Specialization
No successful team tried to make one agent "do everything." All mature cases used role-based division:
- Data ingestion → analysis → report generation → QC
- Director → screenwriter → storyboard → post-production
- Sales assistant → presales → insight report → QC

**Core principle:** A single lobster with an excessively long context degrades. Splitting roles dramatically improves stability.

<div class="zh-trans">所有成熟案例都采用角色分工，没有一个是让单个龙虾"万能干"。核心规律：单只龙虾上下文太长→退化，按工种拆分后稳定性大幅提升。</div>

### 2. MCP Data Integration
MCP connections to 秒针 (Miaozhen) and 魔方Pro were near-universal in marketing teams. Browser Relay (Chrome CDP) was used by multiple teams to bypass anti-scraping barriers.

### 3. SOP Iteration Loop
Every team implemented: lobster makes mistake → record in Memory/AGENTS.md → build benchmark set → score → iterate. The result: systems that improve automatically with use.

### 4. Human Confirmation Gates
High-stakes operations (tax filing, mass messages, client deliverables) kept a human confirmation step. This reflects a healthy human-machine boundary.

### 5. Scheduled Tasks
Converting lobsters from "tools" into "employees" depends critically on scheduled/triggered execution. Nearly all automation projects relied on it.

## Standout Approaches

### Skill Factory (DPA团队 · 史佳艳)
Introduced "小准" — an agent that auto-generates acceptance criteria and scoring rubrics. Workflow: Benchmark set → lobster writes Skill → 小准 scores → iterate. A Skill written by one lobster scored 93/100 on a different machine with zero modification.

<div class="zh-trans">引入"小准"——自动出验收标准和打分的 Agent。龙虾写的 Skill 移植到另一台机器得 93 分，无需修改。人类只需提供 Case 集 + 标准答案。</div>

### TVC Production (区域组华南 · 戈琛)
3 humans + 8 agents produced a Gillette TVC in 7 working days vs. traditional 1–2 months. Reached 80–90% of traditional production quality. Already paid by real clients (Hong Kong/Taiwan). Cost: replaces tens of people with 3 humans + 8 lobsters.

### Automated Tax Filing (财务服务部 · 吴倩)
Lobster logs into the national tax system via Chrome CDP (not traditional RPA). The tax authority cannot detect it as a bot. Handles corporate tax declarations end-to-end, with human confirmation only at the final step. Future applicability: HR individual tax, business registration, statistical reports.

### KOL Risk Screening (海外产品部 · 汤旭耀)
Covers 8 countries, 1,000+ risk terms across 8 risk categories per market. Deployed in production for Universal Entertainment and 欧莱雅. Cost: dropped to 1/10 of traditional vendor. Technical highlights: checkpoint-resume mechanism for long tasks + dual-channel communication (primary + Telegram fallback).

### One-Person Agency (渠道团队 · 李俊杰)
Three lobsters simulate a complete agency: 妙蛙种子 (S4.6, strategy/buying) + 小火龙 (Kimi2.5, market/PM) + 杰尼龟 (千问3.5, creative/social). From RFP brief to full proposal: ~4 hours. Total cost ~¥1,000. Traditional equivalent: 3–4 weeks + much higher labor cost.

### Cross-Machine Agent Collaboration (区域组华北 · 张新颖)
Built a platform for lobsters on different machines to collaborate — assign tasks, report progress, surface issues. Conceptually: an "AI Tieba" as a third-party public lobster coordination layer. 辉哥 noted strong alignment with the OTGO octo strategy.

## Recurring Bottlenecks

### High-Frequency (5+ teams)

| Bottleneck | Manifestation |
|-----------|---------------|
| **Context degradation** | After long tasks, lobster forgets instructions or self-modifies agreed logic |
| **Silent stop / heartbeat failure** | Lobster stops without reporting; task disappears |
| **Token cost** | Complex tasks expensive; KOL screening team spent $5,000 already |
| **Anti-scraping blocks** | 魔方Pro, 小红书, Boss直聘 all block scraping |
| **Information silos** | Multiple agents producing inconsistent data |

### Solved Bottlenecks (Reference Solutions)

| Problem | Solution | Source |
|---------|----------|--------|
| Government site bot detection | Chrome CDP (replaces RPA) | 财务报税组 |
| Context degradation in long tasks | Checkpoint-resume + pipeline file | 网红背调组 |
| Multi-agent communication timeout | Dual-channel (primary + Telegram) | 网红背调组 |
| Skill portability across lobsters | Benchmark + 小准 scoring + Skill export | DPA组 |
| Data bus / information silos | Async-tag-driven unified data package | 明星评估组 |
| 魔方Pro anti-scrape | Browser Relay (CDP) | 华东一账户组 |

## 辉哥's Core Observations

1. **Taste beats tools** — lobster capability is generic; your standard for "what good looks like" is the moat.
2. **Feedback loops drive collective progress** — a Skill shared without receiving back downstream feedback produces no compounding.
3. **Collective intelligence beats solo breakthroughs** — 5 teams doing GEO should pool token efficiency benchmarks.
4. **Input quality is the real asset** — what you feed the lobster (cases, aesthetic judgment, context) is your core IP.
5. **octo is the OS connecting all global lobsters** — internal launch target: April 15, 2026.

<div class="zh-trans">辉哥核心点评：① Taste 比工具更重要；② 反馈闭环是集体进步的关键；③ 集体智慧胜于单点突破；④ 你给龙虾的 Input 才是真正的核心资产；⑤ 开源 octo 是连接全球龙虾的操作系统，目标 4月15日内部上线。</div>
