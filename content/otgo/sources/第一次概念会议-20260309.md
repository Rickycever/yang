## Key Points

### 1. octo — Product Definition

octo (八爪鱼) is an open-source enterprise AI collaboration platform. Its core purpose: connect individual 龙虾 agents into a networked organization with **globally persistent memory**.

<div class="zh-trans">octo（八爪鱼）是开源的企业 AI 协作平台，核心目的是将独立的龙虾 Agent 串联成网络，并实现**全局持久记忆**。</div>

Key differentiator vs. DM (the previous product):
- DM: memory isolated per session — each conversation starts fresh
- octo + 龙虾: memory persists globally across sessions — the more you use it, the better it knows you

Other capabilities: cross-device agent collaboration · private deployment (data stays with client) · MCP GUI support (handles tasks requiring graphical interfaces)

**Status as of 2026-03-09:** Internal testing; basic IM connection capability launching mid-March.

### 2. Dual Sales Strategy

| Client Need | Solution | Follow-up Path |
|-------------|----------|----------------|
| Wants a **platform** | Sell octo (open source, full private deployment, client owns all data and code) | Add agents on top of octo |
| Wants a **point solution** | Sell standard agents (妙/Miao, IDEF, GUO, etc.) | Introduce octo alongside; eventually connect agents to octo |

The two paths are not in conflict — they converge at octo.

<div class="zh-trans">两条路径最终都汇向 octo：客户要平台→直接卖 octo；客户要单点→先卖标品 Agent，顺带介绍 octo，之后把 agent 接入 octo 平台。</div>

**Competitive advantage vs. 飞书/企微/钉钉:**
- Open source + private deployment → data not monopolized by big tech
- Small team + 龙虾 development speed already exceeds 飞书's 5,000-person team
- 飞书 can't open-source (5,000 employees would become redundant)

### 3. Commercial Model — From Man-Hours to Tokens

| Old Model | New Model |
|-----------|-----------|
| Bill by man-days (人日) | Token consumption × management multiplier (×1.1–2) |
| Human labor as the unit | Lobster delivers; humans only intake requirements and verify output |
| Constrained by headcount | "Engineers are unlimited" (infinite lobsters) |

This reframes the entire outsourcing industry: OTGO becomes a **token distributor** that operates with a small human team orchestrating an unlimited agent workforce.

<div class="zh-trans">商业模式重构：OTGO 变成 **token 分发商**，小型人类团队协调无限龙虾交付，按 token 消耗加收管理费，人只管接需求和验收。</div>

### 4. Key Client Situations

**vivo** (邱凌燕)
- AI mid-platform connecting 4 brand lines (UX comms / PR / digital marketing)
- Needs: Content Agent (KOL style replication, QS shopping guide, QL content creation) + media campaign full-funnel
- Final goal: an agent OS connecting first-party data + third-party platforms
- Decision: launch single Content Agent first → connect to octo platform

**顺丰** (张晓/Amy)
- Full-funnel marketing platform: Social Listening → content/script generation → TVC + creative production → ad placement → analytics
- Requires private deployment; budget ~¥1M for POC
- Gap: current DM only covers insight→strategy; production and placement missing
- Decision: 3月出 demo (辉哥保底), then connect to octo

**两江文旅** (王佼佼)
- Budget ¥10M+; build "文旅大脑" (cultural tourism brain)
- 7×24 monitoring of Chongqing cultural tourism dynamics
- Lobster Innovation Competition: 100 groups × 100 lobsters, all connected to octo
- Decision: octo as technical foundation

**星巴克** (张刚)
- Budget ~¥2M; requirements similar to octo use case
- Currently building 龙虾 skill sets for 德克士, 永和 (restaurant industry)
