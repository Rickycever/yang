# 数据反馈优化大模型的数据流(深度展开)

> 配合主报告《Meta 广告平台与 AIGC 创意生成》第 4.1 节"飞轮 A:创意模型微调"使用。本文给出端到端的具体数据流、字段、pipeline 细节,工程团队可直接据此落地。

---

## 一、整体数据流(端到端)

飞轮 A 的核心循环是:**广告投放 → 表现数据回流 → 训练样本构造 → 模型微调 → 新创意生成 → 再投放**。具体到工程上,需要打通 6 个环节:

```
[1] 广告投放(Meta Marketing API)
        ↓ CAPI 事件 + Creative ID 关联
[2] 表现数据采集(CAPI + Impression Pixel + 内部埋点)
        ↓ 实时 ETL
[3] 数据仓库(特征库 + 创意库 + 表现库 三表 join)
        ↓ 离线调度 / Airflow / DolphinScheduler
[4] 训练样本构造(过滤、标注、采样、split)
        ↓ 触发器(新素材积累到阈值 或 周期触发)
[5] 模型微调 pipeline(LLM / Diffusion / Video Gen / 数字人 各一条)
        ↓ 实验平台(MLflow / W&B)
[6] 模型评估 + 灰度上线(A/B 实验 + 业务指标校验)
        ↓ 蓝绿发布
[7] 新模型服务化(替换生产模型的权重 / LoRA adapter)
        ↓ 回到 [1]
```

**关键设计原则**:
- 训练数据必须能**追溯到具体的 Creative ID 和 Ad ID**,否则无法归因到具体素材
- 反馈回路必须**有时效性约束** — 高表现素材从投放到回流到训练,最好不超过 7 天
- 微调要**多任务联合** — 文案模型(图文/视频脚本)+ 视觉模型(产品图/场景图)+ 数字人模型(口播)各走各的,不要一个大模型打天下

## 二、数据采集层:具体字段

### 2.1 CAPI 事件结构(广告主服务端回传)

通过 Conversions API 回传到 Meta 的标准事件,必须包含以下字段以便后续 join:

| 字段 | 类型 | 用途 | 备注 |
|---|---|---|---|
| `event_name` | enum | Purchase / Lead / AddToCart / ViewContent | 区分转化漏斗 |
| `event_time` | int(Unix) | 事件时间戳 | 必须是**服务端时间**,不是客户端 |
| `event_id` | string(uniq) | 事件唯一 ID | 用于去重(和 Pixel 对比) |
| `event_source_url` | string | 触发事件的 URL | 用于归因到具体落地页 |
| `action_source` | enum | website / app / physical_store | 区分来源 |
| `user_data` | object | 用户标识(去标识化) | email_sha256 / phone_sha256 / external_id |
| `custom_data` | object | 业务数据 | value / currency / content_ids / content_type |
| `attribution_data` | object | Meta 归因结果 | ad_id / adset_id / campaign_id |

**最关键的是 `attribution_data` 里的 ad_id** — 它关联到具体投放的 Ad,Ad 关联到具体 Creative,Creative 关联到具体素材。这是把"转化数据"和"创意数据"绑定的唯一锚点。

### 2.2 内部埋点(广告主侧)

除了 CAPI,广告主还应该在自己的网站/APP 内部埋点采集以下数据:

| 字段 | 用途 |
|---|---|
| 用户 session 行为序列(浏览 → 加购 → 下单) | 漏斗分析 + 用户分群 |
| 落地页停留时长 | 评估素材-落地页匹配度 |
| 视频完播率(秒级) | 评估 Hook(前 3 秒吸引力) |
| 多触点归因数据 | 跨 Meta / Google / TikTok / 巨量 |
| 创意特征标注(主体/场景/色调/构图/卖点) | 构造"特征 → 表现"映射 |

**注意**:内部埋点要带一个 `creative_id` 字段(从 URL 参数 utm_content 或落地页 URL 反推),否则和 CAPI 数据 join 不到一起。

### 2.3 创意特征抽取(关键!)

每一条投放的素材,在入库时必须自动抽取结构化特征。常见特征分三类:

**视觉特征**(用 CLIP / DINOv2 / 自训练模型抽取):
- 主体类型(产品图 / 场景图 / 人物图 / 数字人口播)
- 主体占比、构图(三等分 / 中心 / 对角)
- 色调(暖 / 冷 / 中性)、饱和度、对比度
- 背景复杂度(简单 / 中等 / 复杂)
- 是否包含人脸、是否包含文字、是否包含 Logo

**文案特征**(用 LLM 抽取):
- 卖点类型(功能 / 价格 / 情感 / 社会认同)
- CTA 类型(立即购买 / 了解更多 / 限时优惠)
- 文本长度、关键词
- 情感极性(正面 / 中性 / 负面)

**元数据**(人工标注 + 业务系统):
- 行业(美妆/食品/3C/汽车/教育/...)
- 产品类目(SKU 级)
- 目标受众(年龄段/性别/地域)
- 投放时段、版位
- 创意生成方式(AIGC 生成的 / 人工设计的 / 模板生成的)

把这三类特征存到 `creative_features` 表,用 `creative_id` 关联到 `ad_performance` 表,后续做"特征 → 表现"映射查询时用。

## 三、训练样本构造(最容易做错的一步)

### 3.1 样本过滤规则

不是所有投放数据都适合做训练样本。直接拿全量数据训练,模型会学到大量噪声。过滤规则:

**必须剔除的样本**:
1. **预算烧光提前下线的素材** — 没跑出真实表现,样本有偏
2. **冷启动期(0-24 小时)的素材** — Meta 算法还在学习,数据不稳
3. **A/B 测试中失败的素材** — 用了非主流量,数据不代表真实表现
4. **特殊节日/事件素材**(双 11、618) — 表现受外部因素干扰,样本不可泛化
5. **违规被下架的素材** — 不能用作训练数据(法律 + 平台规则)
6. **转化次数 < 100 的素材** — 样本不足,统计噪声大(除非是冷启动必须)

**必须加权的样本**:
- 长期跑量稳定的素材(投了 > 7 天,日均消耗稳定的)
- 大盘表现好的素材(CTR / IPM / Hook Rate 高于行业中位数)
- 不同版位都有覆盖的素材(避免单版位过拟合)

### 3.2 标签设计

监督学习的标签决定了模型学什么。常见标签体系:

**主标签(回归 / 排序目标)**:
- `ctr_3s` — 3 秒可见点击率(去除掉"误触")
- `hook_rate_3s` — 3 秒可见率(评估前 3 秒吸引力)
- `ipm` — 千次曝光转化
- `cvr` — 转化率(转化 / 点击)
- `cpa` — 单转化成本(越低越好,做 reward 而不是 loss)
- `roas` — 广告支出回报率

**辅助标签(分类 / 多任务)**:
- 是否通过 Meta 审核(`is_approved`)
- 是否在 24 小时内被系统自动扩量(`is_auto_scaled`)
- 用户停留时长分桶(`dwell_time_bucket`)
- 转化漏斗位置(`funnel_position`: 浏览/加购/下单/复购)

**负样本标签(必须!)**:
- "差素材"的样本比"好素材"更重要,模型才能学会避免什么。给"被自动缩量的素材"打 `is_penalized` 标签。

### 3.3 数据集划分

| 集合 | 占比 | 用途 | 划分原则 |
|---|---|---|---|
| 训练集 | 70% | 模型训练 | 随机采样 |
| 验证集 | 15% | 超参调优 + 早停 | 按时间切分(后 15%) |
| 测试集 | 15% | 离线评估 | 按时间切分(最后 15%) |

**关键**:验证集和测试集必须**按时间切分**,不能随机切。广告数据有强烈的时序特征(季节性、平台算法演进),随机切分会导致"数据泄漏",模型在测试集上看着很好,线上完全不能用。

### 3.4 反馈时效性

| 素材类型 | 反馈周期 | 建议 |
|---|---|---|
| 短视频(Reels / TikTok) | 7-14 天 | 短视频完播率需要积累 |
| 图文(Facebook Feed) | 3-7 天 | 转化漏斗短 |
| 数字人口播 | 14-30 天 | 数字人接受度需要长周期验证 |
| DPA 目录变体 | 1-3 天 | 单 SKU 表现快速收敛 |

建议 pipeline 至少**每 7 天**触发一次训练,每 30 天做一次大版本迭代。

## 四、模型微调策略(按素材类型拆)

不同类型的素材,微调的模型和策略完全不同。**不要用一个大模型打天下**。

### 4.1 文本创意(广告文案 / 标题 / CTA)

**基础模型**:
- 中文:通义千问 Qwen3、DeepSeek V3、文心一言 4.0
- 英文:Claude Sonnet 4.5、GPT-5、Gemini 3

**微调方法**:
- **LoRA / QLoRA**(推荐)— 8B-70B 级别模型,1-2 张 A100/H100 即可
- **DPO / IPO**(强烈推荐)— 用"好文案 vs 差文案"对,模型直接学"哪个更好"
- **Full Fine-tuning** — 仅在数据量极大(>100K 高质量样本)时考虑

**训练数据格式**:
```json
{
  "prompt": "为美妆品牌 XX 写一条 Reels 短视频脚本,卖点是'24h 持妆',目标受众 25-35 岁女性",
  "chosen": "{\"hook\": \"早 8 点的地铁,你还在补妆?\", \"body\": \"...\", \"cta\": \"立即下单\"}",
  "rejected": "{\"hook\": \"新品上市\", \"body\": \"...\", \"cta\": \"点击购买\"}"
}
```

**评估指标**:
- 自动:BLEU / ROUGE / BERTScore(参考用,不直接代表业务效果)
- 业务:用验证集文案做 AB 实验,看 CTR / CVR 提升
- 人工:每版本抽样 50 条,投放团队 1-5 分打分

### 4.2 视觉创意(产品图 / 场景图 / 数字人背景)

**基础模型**:
- Midjourney V7、DALL-E 4、Imagen 4、Stable Diffusion 3.5
- 国内:可灵、即梦、文心一格

**微调方法**:
- **DreamBooth / LoRA**(单 SKU 风格)— 给特定品牌 / 产品练专属风格
- **IP-Adapter / Reference-only** — 保留参考图风格但换主体
- **ControlNet** — 严格控制构图(产品图必带 Logo 区域)
- **Inpainting / Outpainting** — 给现有素材做局部修改(扩图、换背景)

**训练数据格式**:
```json
{
  "reference_image": "products/skincare_001.png",
  "controlnet_condition": "depth_map.png",
  "prompt": "日系小清新场景,木桌背景,自然光,产品居中,右下角留 Logo 区域",
  "output_style": "minimalist_lifestyle",
  "ctr": 0.034
}
```

**关键**:必须保留产品的**Logo / 包装 / 文字区域**,否则下游审核会被打回。ControlNet 的 depth map 或 canny edge 是必须的。

### 4.3 视频创意(短视频 / 数字人口播)

**基础模型**:
- Sora 2、Veo 3.1、可灵 1.5、Pika 2.0
- 国内:即梦、可灵、智谱清影

**微调方法**:
- **Motion LoRA** — 学特定动作风格(口播手势、产品展示动作)
- **Camera Control** — 控制镜头运动(推/拉/摇/移)
- **Lip Sync Adapter** — 数字人专用,唇形和音频对齐
- **Audio-Driven Generation** — 音频驱动视频(给一段 TTS,自动生成数字人口播视频)

**训练数据格式**:
```json
{
  "audio": "tts/female_voice_01.wav",
  "reference_frame": "digital_human/idol_001/frame_000.png",
  "motion_template": "speaking_gesture_v2",
  "script": "这款精华液的核心成分是...",
  "output_dur": 15.0,
  "hook_rate_3s": 0.42
}
```

**关键**:数字人训练要避免**真人脸 / 明星脸**,用全虚拟形象或明确授权的数字人,否则版权和肖像权风险极大。

### 4.4 多模态统一(进阶)

如果团队有足够算力,可以做一个**多模态统一模型**:
- 输入:产品图 + 卖点文本 + 目标受众 + 历史表现数据
- 输出:完整的创意方案(脚本 + 关键帧 + CTA + 数字人形象)

技术路径:LLaVA / Qwen-VL / InternVL 等多模态 LLM + 工具调用(调用 Midjourney / Sora 等生成模型)。但这个方向投入大、ROI 不明确,建议在飞轮 A 跑通 6 个月后再考虑。

## 五、评估与回流验证(模型不能"上线就完事")

### 5.1 离线评估

| 指标 | 目标 | 计算方式 |
|---|---|---|
| CTR 预测准确率 | > 行业基线 10% | 用测试集计算 MAE / RMSE |
| CVR 预测准确率 | > 行业基线 10% | 同上 |
| 创意多样性 | 多样性指数 > 0.7 | 跨条目的 embedding 余弦相似度 |
| 合规通过率 | > 95% | Meta 审核通过率 |
| 品牌一致性 | > 0.85 | 视觉风格 embedding 与品牌指南相似度 |

### 5.2 在线 A/B 实验

**核心原则**:每个新模型版本必须经过线上 A/B 才能全量。配置:
- **实验组**:新模型生成的素材,5-10% 流量
- **对照组**:旧模型 / 人工设计的素材,5-10% 流量
- **观察期**:7-14 天(短视频)或 3-7 天(图文)
- **显著性**:p < 0.05,样本量 > 1000 次转化

**业务指标校验**:
- 主指标:CPA / ROAS(必须正向提升 > 5%)
- 辅助指标:CTR / Hook Rate / 创意审核通过率
- 防退化指标:违规率、品牌一致性、用户负面反馈率

### 5.3 防"reward hacking"

模型可能学会"作弊"来提高表现指标,导致线上效果反而下降。常见反模式:

1. **夸张标题党** — 模型学到"震惊!"、"必看!" 等词能提高 CTR,但损害品牌
2. **诱导点击** — 模型学到"美女图 + 模糊文字"能提高点击,但转化率暴跌
3. **短期 ROI 优化过拟合** — 模型只学"短期 CPA 低的素材",但损害长期 LTV

**防范方法**:
- 多任务联合训练,加约束项(品牌一致性、违规率)
- 定期人工抽检(每周 20 条新素材)
- 长周期评估(不只是 7 天,要看 30 天 LTV)
- 设置硬约束(违规率 > 阈值直接回滚)

## 六、冷启动与规模化路径

### 6.1 冷启动(0-3 月):没数据,怎么办

新广告主 / 新产品没有历史数据,飞轮起不来。冷启动方案:

1. **用行业 benchmark 数据**(AdCreative.ai、巨量案例库、蓝标公开数据)做 Prompt 模板的初始值
2. **少量素材 + 高预算快速验证** — 投 50-100 条素材,每条投足 500-1000 美元,7 天内拿到第一批可靠表现数据
3. **迁移学习** — 从类似行业的"老客户"模型迁移(注意数据隔离和合规)
4. **规则化生成 + LLM 润色** — 前 3 个月用规则模板 + LLM 润色,不要直接上 AIGC 微调

### 6.2 规模化(3-12 月):飞轮开始转

- 第 3-6 月:Prompt 模板库扩展到 20+ 行业,每月新增 1000+ 训练样本
- 第 6-9 月:开始微调第一个 LLM(文本创意),同时建立特征库
- 第 9-12 月:微调第一个 Diffusion(视觉创意),飞轮 A 正式跑通

### 6.3 关键里程碑

| 月份 | 训练样本量 | 微调模型 | 业务影响 |
|---|---|---|---|
| 0-3 | 0(冷启动) | 第三方 API(Midjourney/Gemini) | 创意产能 +50% |
| 3-6 | 1K-10K | 规则模板 + LLM 润色 | 创意产能 +200% |
| 6-9 | 10K-50K | 第一个 LLM 微调(文本) | 文案 CTR +10-20% |
| 9-12 | 50K-200K | 第一个 Diffusion 微调(视觉) | 视觉 CVR +15-30% |
| 12-18 | 200K+ | 多模态统一 / 数字人 | 整体 ROAS +30-50% |

## 七、常见坑(过来人的血泪)

### 7.1 数据漂移

Meta 算法在持续演进,2025-2026 已经过 Andromeda → GEM → Advantage+ Framework 三次大改。每次大改,老训练数据可能"过期"。

**对策**:
- 每季度重训一次,加入新平台数据
- 用 embedding 可视化监控数据分布漂移
- 设置数据新鲜度指标(< 30 天的样本占比)

### 7.2 Selection Bias(选择偏差)

只有"被系统选中投放的素材"才有表现数据,但模型需要的是"所有可能素材"的表现分布。

**对策**:
- 主动投一些"探索性"素材(epsilon-greedy),不是为了拿短期 ROI,是为了拿训练数据
- 维护一个"探索预算"(总预算的 5-10%)

### 7.3 反馈延迟

短视频广告的完整表现需要 14-30 天才能稳定。但模型训练不能等这么久。

**对策**:
- 用"早期信号"(3-7 天的 Hook Rate / CTR)做粗排,用"晚期信号"(14-30 天的 ROAS)做精排
- 多阶段训练:先用早期信号训一个模型,再用晚期信号 fine-tune

### 7.4 创意多样性坍塌

模型训练后,所有生成的素材都越来越像(过拟合到历史高表现素材)。

**对策**:
- 训练数据加多样性约束
- 推理时用 temperature > 0.7 采样
- 定期人工注入"创意种子"打破模式
- 监控 embedding 聚类,看是否坍塌到单一 cluster

### 7.5 平台合规突变

Meta 政策经常更新,2026 年 1 月起就取消了 7 天/28 天观看归因窗口(待官方二次确认)。

**对策**:
- 监听 Meta Business Help Center + Marketing API Changelog
- 飞轮的"归因"部分做成插件化,平台规则变了能快速切换
- 不要把"平台归因数据"当作"真实增量"的唯一来源

## 八、给工程团队的实施清单

按优先级排序,前 3 个月必做:

- [ ] **CAPI + 内部埋点 + Creative ID 三表 join 跑通** — 没有这个后面都是空话
- [ ] **创意特征库 v1** — CLIP + 规则化抽取视觉特征,LLM 抽取文案特征
- [ ] **数据仓库分层** — ODS(原始 CAPI/Pixel)→ DWD(join 后)→ DWS(按素材聚合)→ ADS(给训练用)
- [ ] **离线调度** — Airflow / DolphinScheduler,每天 ETL,每周训练数据刷新
- [ ] **Prompt 模板库** — 5-10 行业,每个行业 20-50 个高质量模板
- [ ] **AB 实验平台** — 支持流量切分、显著性检验、自动 winner 决策
- [ ] **评估指标 dashboard** — CTR / CVR / ROAS / 合规率 / 品牌一致性 实时监控

3-6 个月必做:

- [ ] **第一个 LLM 微调(文本)** — Qwen3-8B + LoRA,3-5K 高质量样本起步
- [ ] **DPO 训练** — 用"好文案 vs 差文案"对
- [ ] **线上 A/B 平台打通** — 新模型生成素材 → AB 实验 → 数据回流 → 下一轮训练
- [ ] **特征 → 表现 映射查询** — 支持"什么特征组合的素材表现最好"反向查询

6-12 个月必做:

- [ ] **第一个 Diffusion 微调(视觉)** — SD 3.5 + LoRA
- [ ] **数字人模块** — 用全虚拟形象,避免版权风险
- [ ] **多任务联合训练** — 文案 + 视觉 + 数字人协同优化
- [ ] **行业 benchmark 订阅** — 买第三方数据(AdCreative.ai、巨量案例库)做冷启动和参考

---

## 一句话总结

**飞轮 A 的本质是"用广告表现数据做有监督的 AIGC 创意模型微调",工程上的难点不是模型本身,而是"数据采集 → 训练样本构造 → 线上 AB 评估"的闭环,以及冷启动、多样性、防退化这些"软工程"问题。建议先把 4 个组件(数据仓库 / 特征库 / AB 平台 / 评估 dashboard)搭起来,3 个月后再启动第一个 LLM 微调,不要一上来就上 Diffusion / 数字人。**
