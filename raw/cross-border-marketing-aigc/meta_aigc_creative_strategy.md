## 一、Meta广告平台核心规格

### 主要广告格式

| 格式 | 核心规格 | 适用场景 |
|---|---|---|
| 单图片广告 | 1080×1080（1:1）/ 1080×1350（4:5）/ 1200×628（横向） | Feed全版位 |
| 视频广告 | 9:16竖屏1080×1920，MP4/MOV，H.264，≤4GB | Reels/Stories/Feed |
| 轮播广告 | 2-10张，每张1080×1080，统一比例 | 多产品展示 |
| 精品栏 | 封面图/视频+商品图，需配合商品目录 | 电商转化 |
| 即时体验 | 全屏移动端落地页，支持视频+轮播+表单 | 品牌旗舰/新品发布 |
| Advantage+动态创意 | 上传多版素材，AI自动组合测试 | A/B测试优化 |

### 关键技术规则

- **图片文字占比 ≤ 20%**，超出会被系统降低投放量
- **Reels安全区**：顶部250px + 底部340px 不放关键信息（UI遮挡）
- **视频黄金3秒**：前3秒必须抓眼球，无声状态下也要能传达核心信息
- **before/after对比**：明确禁止（医疗/减肥类尤其严格）
- **AI生成人脸**：需强制标注，品牌方普遍不接受

### 2025年平台新动向

- **Andromeda算法**：创意内容本身变成定向信号，创意质量和多样性直接影响受众覆盖
- **Advantage+ Sales Campaign**：A+SC全面升级，AI自动分配新客/老客预算
- **Meta AI创意工具**：背景图生成、视频生成2.0、AI配音/翻译（9种语言）
- **品牌一致性控制**：上传Logo/色彩/字体，AI保持创意风格统一

## 二、AIGC产品定位与差异化

### 核心定位

> **"面向Meta投放团队的品牌级广告创意工作站"**  
> 不是AI生图工具，是让投放团队自己能出图、能改图、能直接投的生产系统

**目标客户**：国内品牌的Meta投放团队 + 管理多品牌客户的中型Agency  
**核心痛点**：出图速度跟不上投放节奏、素材量不够支撑A/B测试、跨品牌管理混乱

### 为什么不做"全链路"

0→1阶段做全链路的结果是每个环节都是二流，没有一个环节能让用户形成依赖。需要先选一个**用户每天必须回来用的锚点动作**，其他功能围绕它展开。

### 核心差异化功能（按优先级）

**P0：品牌调性锁定 + 合规生成**
- 构建品牌知识库（色值、字体、视觉风格、调性词库）
- 生成结果始终在品牌框架内，而非通用AI风格
- 内置Meta合规检测：图片文字占比、安全区标注、内容敏感词扫描
- **核心价值**：生成即合规，上传即通过

**P0：多尺寸一键导出**
- 上传一张主图，智能识别视觉重心，自动裁切+补全
- 输出四个规格：1:1 / 4:5 / 9:16 / 1.91:1
- **核心价值**：投放团队每周省3-5小时，最高频刚需

**P1：变体矩阵生成（结构化，不是随机）**
- 固定产品主体，变化：背景场景 × 文案切角 × 视觉风格
- 输出可解释的变体表格，每个变体对应一个测试假设
- 直接对接Meta Dynamic Creative投放逻辑

**P1：效果归因反哺**
- 接入Meta Marketing API，拉回素材维度效果数据（CTR/CPM/频次）
- 数据与生成记录关联，AI逐步学习"什么对这个品牌有效"
- **核心价值**：从工具到平台的关键跨越，构建数据飞轮

**P2：投放前审核协作流**
- 产品内流转审核：标注修改意见 → 在线修改 → 一键推送Meta Business Manager
- 把产品从"生成工具"变成"工作流系统"，显著提高客户粘性

### 竞品格局判断

| 竞争对手类型 | 他们做什么 | 你的优势 |
|---|---|---|
| 纯AI工具（AdCreative.ai等） | 通用模型，无品牌深度 | 品牌化+供应商网络兜底 |
| 平台内置AI（Meta Advantage+） | 投放系统内优化，无独立生产能力 | 工作流+多客户管理 |
| 传统创意Agency | 高质量但低效高成本 | 速度和规模化能力 |
| **最危险**：有技术能力的大Agency自建系统 | 和你做同样的事 | 需在12个月内形成对外服务版本 |

## 三、创意生产链路与AI介入节点

### 完整流程与AI替代程度

```
策略层（人工主导，AI辅助）
├── 品牌briefing          → 品牌方主导，无法替代
├── 创意策略/竞品分析      → AI可做竞品监测、趋势研究
└── 创意brief撰写         → AI可辅助文案初稿

生产层（AI高度可替代，外包成本集中区 ~65%）
├── 文案撰写              → AI可高度替代（多切角批量生产）
├── 主视觉设计            → AI辅助参考图生成，人工定稿
├── 多尺寸适配            → ★ AI完全替代，最高优先级
├── 批量变体生产          → AI可高度替代（换背景/文案/色调）
└── 修改返稿（平均3-5轮） → 调参替代改稿，周期从天压缩到分钟

审核+投放层
├── 内部审核              → AI做合规预检，人工最终确认
├── 品牌方审核            → 人工主导，产品内在线协作减少摩擦
├── 上传Meta后台          → API一键推送，替代手动操作
└── 效果分析反哺          → AI归因分析，反哺下次创意策略
```

### 外包成本结构

| 工作类型 | 成本占比 | AI替代难度 | 优先级 |
|---|---|---|---|
| 执行类（改尺寸/切图/加字幕） | ~35% | 极低，可立即替代 | P0 |
| 批量生产类（变体/节日版本） | ~30% | 低，建模后替代 | P0 |
| 修改返稿类 | ~15% | 中，流程重设计 | P1 |
| 脚本文案类 | ~12% | 中，需品牌训练 | P1 |
| 策略创意类 | ~8% | 高，暂不替代 | P3 |

**关键结论**：P0两项（执行+批量）合计~65%，是实现30%总成本下降的主战场。P0替代率达50%，总成本降幅已超30%目标。

### 广告文案AI生成的能与不能

**能做好的**：
- 批量变体生成（10-20个不同切角版本）
- 标准化促销/节日文案
- 多切角快速覆盖（Meta Dynamic Creative需要的"量"大于"质"）

**有限制的**：
- **品牌语气细腻度**：AI默认"通用广告腔"，需要大量历史文案做few-shot训练
- **Meta平台特殊性**：前125字符必须出现核心信息（移动端折叠），AI不了解平台转化机制
- **监管敏感类目**：母婴/医疗/金融类有严格文案规则，AI不知道会生成被拒审内容
- **本地化语境**：出海场景的语言习惯，机翻质量不够

**产品设计建议**：
- 建立**品牌文案记忆库**，从历史高CTR文案提取语气和句式
- 内置**Meta文案合规检测**，生成后自动扫描绝对化用词/敏感表达

## 四、视频创意的AI解法

### Reels广告叙事结构

| 时间段 | 功能 | AI能力 | 合规红线 |
|---|---|---|---|
| 0-3秒 Hook | 决定是否划走 | 可生成文字hook脚本 | 真人出镜AI暂无法替代 |
| 3-8秒 建立兴趣 | 呈现核心卖点 | 可生成脚本+字幕 | before/after明确禁止 |
| 8-20秒 建立信任 | 打消顾虑/建立认知 | 可生成脚本框架 | 虚构用户评价违规/AI脸需标注 |
| 20-30秒 CTA | 驱动行动 | 高度可替代文案+排版 | — |

### 视频生产技术栈

**① 脚本生产（现在可做，成熟度最高）**
- 工具：LLM + 结构化prompt模板
- 输出：时间轴脚本JSON（含每秒画面/旁白/字幕）
- 关键：输出结构化JSON，驱动后续自动化链路

**② 分镜头生成（现在可做，降低拍摄沟通成本）**
- 工具：Midjourney / Flux / DALL·E 3
- 用途：给导演和客户对齐视觉预期，减少拍摄后返工
- 限制：人物跨帧一致性仍有瓶颈

**③ AI视频生成（部分可用，分场景判断）**
- 可用：产品特写动效、背景场景生成、Logo动画、转场
- 不可用：真人出镜（Meta合规+品牌方不接受）
- 工具：Runway Gen-3 / Kling / 即梦（保持可替换，不绑定单一模型）

**④ 素材加工处理（完全可做，ROI最高，立即产品化）**
- 竖屏适配：FFmpeg + SAM2主体检测（防止产品被裁切）
- 自动字幕：Whisper ASR + 时间轴对齐 + 品牌字幕模板
- 多语言配音：ElevenLabs / 火山引擎TTS
- 背景替换：SAM2分割 + AI背景生成

**⑤ 智能剪辑合成（模板化场景自动化，创意剪辑人工介入）**
- 模板驱动：脚本JSON → 自动排列素材 → 渲染输出
- 创意剪辑：AI做粗剪，剪辑师做精剪，工作量减少60-70%

**⑥ Hook版本裂变（最核心差异化，技术可行，市场几乎空白）**
- 原视频3秒后固定，只替换前3秒
- 生成4-6个Hook类型版本（文字冲击/问题悬念/产品特写/场景代入）
- 直推Meta Dynamic Creative自动测试
- 测试数据反哺脚本生成，构建创意飞轮

### 技术路线三步走

```
第一步（立即做）：素材加工处理
→ 竖屏适配/字幕/多比例输出，直接替代外包后期工作

第二步（1-3个月）：脚本生产 + Hook裂变
→ 形成"脚本→视频→测试→反哺"小闭环

第三步（3-6个月）：分镜生成 + 模板合成
→ 链路完整，可对外提供端到端交付

AI视频生成作为增强模块接入，保持模型层可替换
```

## 五、图片 vs 视频：先做什么

### 结论：先做图片，为视频设计好扩展路径

**先做图片的理由**：
- 需求密度高：图片需求是视频的5-10倍（一个品牌每月200张图 vs 8-10条视频）
- 生产链路短：分钟级跑通，视频需要几十分钟
- 品牌调性验证快：生成一张图5秒，立即得到反馈
- 技术风险低：图片生成技术栈完全成熟
- 外包替代效果立竿见影：200家供应商中做图片执行的占多数

**做图片阶段要提前设计好的两件事**：
1. **品牌知识库数据结构**按"图片+视频通用"设计，避免后期重构
2. **文案模块**从图片阶段就要建，图片文案和视频脚本共用同一套生成引擎

**切到视频的时机信号**（不是时间表）：
- 图片模块月活稳定，核心工作流跑通
- 品牌知识库积累了足够的客户数据，AI生成有品牌辨识度
- 通常在图片模块稳定运行3-4个月后

## 六、品牌资产库与脚本模板体系

### 品牌资产库（Brand Kit）四层结构

**L1 视觉规范层**（图片和视频共用）
- 色彩系统：主色/辅色/禁用色，HEX+RGB
- 字体规范：主字体/辅助字体，标题/正文/字幕分别定义
- Logo资产：正版/反白/单色，安全间距规则
- 视觉风格标签：整体调性 + 摄影风格 + 参考图（正例/反例各5张）

**L2 内容规范层**（驱动文案和脚本生成）
- 品牌语气：人格描述（3-5个词）+ 禁用词库 + 推荐表达
- 卖点库：核心卖点优先级排序 + 卖点×受众映射
- 受众画像：核心人群 + 痛点/动机/场景 + 敏感点
- 合规规则：行业禁用词 + Meta政策限制 + 品牌方红线

**L3 素材资产库**（AI生成的原材料）
- 产品图库：白底图（必须）+ 场景图 + 细节特写
- 视频素材库：原始片段 + UGC授权素材（标注时间轴和分类）
- 高效创意档案：历史高CTR图片/视频 + 标注"为什么有效"
- 模板库：图片版式/视频片头片尾/字幕样式/促销贴片

**L4 效果记忆层**（随投放数据积累，自动更新）
- 素材效果数据：CTR/CPM/完播率/转化率
- 创意规律总结：哪类Hook有效/哪种色调转化高
- 下次生成权重调整：自动提高有效创意元素的生成概率

### 品牌资产库的创建方式

**引导式初始化（三步，避免让用户填空白表单）**：
1. **上传触发解析**：上传VI手册PDF或历史广告图，系统自动提取色值/字体/logo，生成预填充档案
2. **对话式补全**：系统提问关键问题（品牌人格/核心受众/禁用词），对话形式而非表单
3. **高效创意标注**：上传5张历史高效素材，系统分析共同视觉特征作为AI正向参考样本

**管理机制**：
- 按客户严格隔离，版本控制必须有
- L4效果记忆层完全自动化（接入Meta API后自动更新，无需人工维护）
- 素材生命周期管理：草稿→投放中→已疲劳→已归档

### 脚本模板体系

**核心设计原则：内容和形式分离**

同一份内容单元（Hook文案/卖点/CTA/视觉场景描述）向两个方向分发：
- 向左 → 图片广告（主视觉区 + 文字层 + 卖点徽章 + CTA区域）→ 输出4个规格
- 向右 → 视频脚本（Hook段0-3s / 兴趣段3-8s / 信任段8-20s / CTA段20-30s）→ 输出JSON时间轴

**三层模板结构**：
1. **叙事框架层**：内置6-8个验证有效的框架（如痛点-解决-信任-行动），与格式无关
2. **内容填充层**：AI从品牌知识库拉取卖点和受众画像，生成具体文案和场景描述
3. **格式渲染层**：内容转化为图片排版指令或视频时间轴JSON（唯一需要维护两套逻辑的地方）

**管理要点**：
- 模板要有效果追踪（哪类品牌用哪个框架效果更好）
- 变体要有亲子关系（追踪原版和所有衍生变体）
- 创意疲劳预警（接入Meta频次数据，CTR下滑时自动触发新变体生成）

## 七、供应商成本优化与业务转型

### 你的真实起点（优势）

- 200+家供应商网络 = 真实的创意生产数据 + 产能网络
- 现成客户关系 = 不需要从零冷启动
- 历史稿件和效果数据 = 训练品牌风格模型的原材料

### 两阶段路线

**第一阶段（0-6个月）：内部工具化，打出30%降本**

三个核心动作：
1. **多尺寸批量导出**：选10-20个高频客户录入品牌资产，尺寸适配不再发给供应商 → 单项节省17%总成本
2. **变体批量生产**：节假日/促销/多SKU版本标准化，供应商只做第一张原稿 → 批量执行由系统完成
3. **修改流程收敛**：在线标注+调参界面，AM直接标注需求，平均返稿轮数从3-5轮压到1-2轮

**第二阶段（6-18个月）：产品化，准备对外服务**

此时你有两个别人没有的资产：
- **真实创意生产数据**：供应商历史稿件 + 客户修改记录 + 投放效果数据
- **供应商网络本身**：从"外包方"变成"平台合作商"，系统处理80%执行工作，供应商承接20%高价值工作

### 对外服务定位

> **"有供应商网络兜底的AI创意生产平台"**  
> 能承诺：48小时内交付符合Meta投放规范的完整素材包，AI做80%，人工审核20%，效果不达标可迭代

**目标客户**：年投放预算500万以上的中大型品牌，或管理10个以上品牌的中型Agency

### 向Agency销售的正确话术

- ❌ 错误："帮你裁人、降低外包成本"（威胁感）
- ✅ 正确："同样的人力，能接更多客户、出更多量"（增收逻辑）

### 创意疲劳管理（容易被忽视）

- Meta核心受众看同一条创意7-10天后CTR快速衰减
- 产品需要**持续的创意供给能力**（每周产出新变体替换疲劳素材）
- 这让产品从"项目工具"变成"订阅服务"，客户粘性和续费率大幅提高

### 补充：五个重要盲区

1. **CAPI数据基础**：iOS政策后纯浏览器追踪丢失40-60%转化数据，效果归因的前提是数据质量，产品需有"数据健康度检测"模块
2. **UGC素材处理**：UGC风格广告往往比专业制作表现更好，采购+AI后期加工是完整链路
3. **结构化创意测试**：固定变量测假设（专测Hook/专测视觉/专测文案），结论可积累复用
4. **Andromeda算法影响**：创意多样性直接决定受众覆盖范围，不只影响CTR
5. **12个月窗口期**：有技术能力的大Agency会自建系统，需在此之前形成对外服务版本

## 八、业务运营诊断框架

### 诊断要回答的三个核心问题

- **钱花在哪里**：200家供应商费用结构，哪类工作占多少比例
- **时间卡在哪里**：从需求到素材可投放，卡点在哪个环节
- **质量差在哪里**：有多少素材经过测试迭代，效果数据有没有反馈闭环

### 五个调研维度

| 维度 | 核心问题 | 数据来源 | 产出 |
|---|---|---|---|
| D1 成本结构 | 费用按类型分布？图片vs视频？执行vs创意？ | 财务系统+供应商合同 | 费用热力图+可替代空间测算 |
| D2 时间效率 | 需求到交付多少天？返修几轮几天？ | 项目管理工具+AM访谈 | 瓶颈环节识别+周期压缩空间 |
| D3 质量效果 | 素材跑多少天？A/B测试比例？拒审率？ | Meta后台数据+投放访谈 | 效果基准线+改进潜力 |
| D4 供应商结构 | 200家按能力分几类？核心供应商几家？ | 采购记录+供应商访谈 | 供应商分层策略+淘汰路径 |
| D5 内部协作 | 需求怎么传达？审核几个层级？素材怎么管理？ | 全角色访谈+工具梳理 | 流程再设计建议 |

**调研优先级**：D1→D2→D4→D3→D5（D1+D2两周内完成即可支撑初稿）

### 必须拿到的三个数字

- **单件成本基准**：一张图/一条视频平均多少钱（按工作类型分类）→ AI替代ROI的分母
- **平均交付周期**：需求确认到素材可投放的中位数天数 → 优化目标的起点基准
- **素材利用率**：生产的素材里真正上线的比例+上线后平均跑多少天 → 生产浪费的量化

### 诊断报告结构

1. **现状快照**：费用分布饼图 + 流程时间轴 + 素材漏斗（三张图，数字说话）
2. **痛点热力图**：问题按"频率×严重程度"矩阵排列，引发各角色讨论
3. **可量化改进空间**：具体到"解决X问题能省Y万元/快Z天"
4. **分阶段行动建议**：每阶段目标可量化，有检验标准

### 容易忽视的调研重点

**摸清"现在谁在做决定"的权力地图**：组织架构图上的权力和实际决策权往往不一致。产品上线如果影响了某个人的决策权或存在感，这个人会成为落地阻力。提前识别，把他们变成盟友。

## 九、项目亏本问题拆解

### 亏本的三层原因

**原因A：定价问题**（卖出去的价格太低）
- 典型表现：销售为拿单压价；创意费打包进投放费用里实际免费；报价没有成本基础
- 诊断问题：报价时有没有成本测算支撑？

**原因B：估算问题**（报价时低估了成本）
- 典型表现：返修轮数没有上限；需求范围被扩大（scope creep）；历史数据不足参考
- 诊断问题：项目结束后实际成本vs估算差多少？

**原因C：执行问题**（执行过程中成本超出）
- 典型表现：返修次数超预期；加急费用未计入；AM协调时间成本没有被定价
- 诊断问题：有没有项目级成本实时追踪？

### 快速定位原因：拉3个亏本项目做解剖

1. 报价单上的创意费是多少？当时怎么算出来的？→ 说不清楚 = 原因A
2. 项目结束后实际花了多少？和报价差多少？→ 超出50%以上 = 原因B
3. 执行过程中有没有计划外费用？谁审批的？→ 有大量无人审批的计划外 = 原因C

### 短期三个动作（不需要AI工具，立即可做）

**1. 建立Work Breakdown & Rate Card**
把创意工作拆成最小计费单元，每个单元定固定价格：
- 单张图片制作（含3轮修改）：X元
- 多尺寸适配（基于主图）：X元/套
- 30秒视频脚本：X元/条
- 第4轮起修改费：X元/次
- 报价直接查表，不靠感觉

**2. 给每个项目设置成本预警线**
- 项目开始时记录：预计范围 + 成本预算
- 每周更新实际已发生成本
- 触达预算70%时：评估剩余工作量，超出则与客户商谈变更

**3. 把"返修轮数"写进合同**
- 明确：基础服务包含X轮修改，第X+1轮起收费Y元
- 修改范围限于原始需求，需求变更另行报价
- 效果：客户会开始想清楚再提需求

### 向销售拉齐认知的方法

不讲道理，讲数字。把三个亏本项目的完整成本明细拆给销售看，然后问：**"你希望我们能接哪种创意项目？"**

- 高定制高返修 → 报价必须覆盖这个成本
- 标准化批量执行 → 可以低价但严格限定范围

**AI工具和成本管控的顺序**：先建立定价机制和成本管控机制，再用AI降低生产成本。顺序反了，省下来的钱会继续被无限返修消耗，亏本不会根本改变。

## 十、供应商报价混乱问题

### 问题本质（竞争对手代理场景）

竞争对手代理公司找同一家供应商，拿到更低报价，信息流传到客户耳朵里。这不是采购管理问题，是**供应商忠诚度和信息安全问题**。

**三个真实危害**：
1. 定价策略被竞争对手摸透，总是比对手贵一点点
2. 供应商掌握谈判筹码，反过来压你的采购价
3. 客户获得持续压价工具，每次续约都拿竞争对手报价来压

### 解决方案：三个层面

**供应商层面——分级管理，核心供应商签排他**

| 分级 | 数量 | 策略 |
|---|---|---|
| 战略级 | 10-20家 | 保证采购额换排他协议，优先付款+优先档期 |
| 合作级 | 中等数量 | 签最惠国条款（给你的价格不高于任何其他客户） |
| 机会级 | 其余 | 按需使用，不涉及敏感业务 |

**信息层面——控制供应商知道什么**
- 订单只告知交付要求，不告知最终客户是谁（用代号）
- 不在供应商面前讨论竞争情况和内部价格策略
- 核心客户项目要求供应商签保密协议

**竞争层面——建立深度绑定**
- **量的绑定**：稳定月均采购额，供应商优先保障你的质量和档期
- **能力共建**：核心供应商优先使用你的AI工具，形成平台依赖
- **信息共享**：分享匿名化效果数据，从纯交易关系变成共同成长

### 最紧迫的一个动作

**找到信息是怎么流出去的，堵住这个口**：
- 供应商主动透露 → 重新评估这段关系，商业边界有问题
- 客户自己去问的 → 说明客户对你的黏性不够，是信任度问题

**根本解法**：提升客户对你的不可替代感，客户觉得你不可替代，就不会有动力做价格比较。
